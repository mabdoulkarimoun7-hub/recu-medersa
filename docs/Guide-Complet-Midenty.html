1. Presentation de l'application

Gestion Medersa est une application mobile (PWA) qui permet aux ecoles coraniques du Niger de :

  • Inscrire des eleves avec un matricule automatique (ex: MEI-001, MEI-002...)
  • Suivre les paiements mensuels avec une grille de mois visuelle
  • Generer des recus en PDF, image PNG, ou les envoyer par WhatsApp
  • Imprimer des recus sur imprimante thermique Bluetooth (58mm)
  • Interface bilingue arabe / francais
Multi-clients : Une seule URL (midenty-medersa.netlify.app) dessert toutes les ecoles. Chaque ecole a sa propre configuration (nom, logo, couleurs, frais) chargee via un code d'acces unique.

Modele commercial

Vente unique (pas d'abonnement). Le client paie une fois, recoit son lien personnalise, et utilise l'application indefiniment. Support via WhatsApp + videos tutos.

4. Comment fonctionne l'application

Le parcours d'un client

  1. Vous creez la configuration de l'ecole dans le panneau admin
  2. Vous cliquez "Enregistrer" — le fichier est envoye automatiquement sur GitHub
  3. Netlify detecte le changement et deploie en ~1 minute
  4. Le panneau affiche le lien personnalise du client (ex: midenty-medersa.netlify.app?c=ABRAR-2026-469)
  5. Vous envoyez ce lien par WhatsApp au client
  6. Le client ouvre le lien, voit le logo et le nom de SON ecole
  7. Il entre son code d'acces (pre-rempli automatiquement)
  8. L'application se charge avec ses couleurs, ses classes, ses frais
  9. Il peut installer l'app sur son ecran d'accueil comme une vraie application

Les 4 onglets de l'application

OngletFonction
InscriptionInscrire un nouvel eleve (matricule automatique), generer un recu d'inscription
ClassesVoir la liste des eleves par classe, exporter en PDF
MensuelEnregistrer les paiements mensuels, selectionner les mois, generer un recu
ParametresGerer les classes, frais, sauvegardes, annee scolaire

6. Le deploiement automatique (GitHub + Netlify)

Comment ca marche ?

Le deploiement est 100% automatique. Voici la chaine :

1. Panneau Admin
→
2. API GitHub
→
3. Netlify
→
4. En ligne !
  1. Vous cliquez "Enregistrer" dans le panneau admin
  2. Le fichier JSON du client est envoye sur GitHub via l'API (avec votre token)
  3. GitHub recoit le fichier dans le dossier clients/
  4. Netlify detecte automatiquement le changement sur GitHub
  5. Netlify reconstruit et deploie l'application en ~1 minute
  6. Le lien personnalise du client fonctionne immediatement

Le token GitHub

Pour que le panneau admin puisse envoyer des fichiers sur GitHub, il a besoin d'un Personal Access Token (PAT).

Important : Le token est stocke dans le navigateur (localStorage) de l'appareil ou vous etes connecte au panneau admin. Si vous changez d'appareil, vous devrez re-saisir le token.

Si le token expire ou ne marche plus :

  1. Allez sur github.com → Connectez-vous
  2. Cliquez sur votre photo (en haut a droite) → Settings
  3. Menu de gauche → Developer settings
  4. Personal access tokens → Tokens (classic)
  5. Cliquez "Generate new token (classic)"
  6. Note : Midenty Admin, Expiration : No expiration
  7. Cochez "repo" (la premiere case)
  8. Cliquez "Generate token"
  9. COPIEZ LE TOKEN IMMEDIATEMENT (il ne s'affiche qu'une seule fois !)
  10. Collez-le dans le panneau admin → section "Deploiement automatique"

Comment ca fonctionne techniquement

  3. Il telecharge le fichier clients/CODE.json depuis le serveur

Dans le panneau admin, deux methodes :

9. Le mode hors-ligne et la synchronisation

Le principe : local + cloud

L'application utilise un systeme hybride intelligent :

  • Les donnees sont stockees localement (localStorage) ET dans le cloud (Firebase Firestore)
  • Quand il y a internet → les donnees se synchronisent automatiquement entre le telephone et le cloud
  • Quand il n'y a PAS internet → l'app continue de fonctionner avec les donnees locales
  • Quand internet revient → tout se synchronise automatiquement, sans rien faire
Telephone perdu ou change ? Pas de panique ! Le client se connecte sur un nouveau telephone avec son code d'acces, et toutes ses donnees (eleves, paiements, historique) se rechargent automatiquement depuis Firebase.

11. Codes secrets et securite

NE JAMAIS PARTAGER CES CODES AVEC LES CLIENTS
CodeValeurUtilisation
Code reset admin MIDENTY-RESET-2026 Reinitialiser les donnees d'un appareil (tap 5x sur "A propos" → saisir ce code)

Regles de securite (non negociables)

  • Pas de suppression d'eleves : Seule la modification (nom, classe) est autorisee. Un eleve qui a paye ne doit jamais disparaitre.
  • Reset en deux etapes : Il faut taper 5 fois sur "A propos", puis entrer le code secret, puis confirmer. Seul Midenty connait le code.
  • Infos ecole verrouillees : Le client ne peut PAS modifier le nom, le logo, ni les telephones depuis l'application. Seul Midenty peut le faire via le panneau admin ou le panneau dev cache.

Le panneau developpeur cache

Dans l'application, onglet Parametres → taper 5 fois sur "A propos" → entrer le code MIDENTY-RESET-2026 → choisir "DEV" au lieu de "RESET". Cela affiche un panneau cache pour modifier les infos ecole directement sur l'appareil.

14. Guide : Comment ajouter un nouveau client

  1. Allez sur midenty-medersa.netlify.app/admin/ et connectez-vous
  2. Cliquez le bouton "+ Nouveau client"
  3. Remplissez les informations de l'ecole (nom FR, nom AR, adresse, logo...)
  4. Choisissez les couleurs du theme (ou gardez les couleurs par defaut)
  5. Le code d'acces est genere automatiquement, ou vous pouvez en saisir un manuellement
  6. Ajoutez les classes (Niveau 1, Niveau 2, etc.)
  7. Definissez les frais (inscription et mensuel)
  8. Cliquez "Enregistrer"
  9. Le fichier est automatiquement envoye sur GitHub → Netlify deploie en ~1 min
  10. Le lien personnalise s'affiche → cliquez "Copier le lien"
  11. Envoyez ce lien au client par WhatsApp

15. Guide : Comment modifier un client existant

  1. Dans le panneau admin, trouvez le client dans la liste
  2. Cliquez "Modifier"
  3. Changez les informations necessaires
  4. Cliquez "Enregistrer"
  5. Le fichier est mis a jour sur GitHub → Netlify redploie automatiquement
Note : Les modifications s'appliquent au prochain login du client. Les donnees existantes (eleves, paiements) ne sont pas affectees.

16. Guide : Comment desactiver un client

  1. Dans le panneau admin, cliquez "Modifier" sur le client
  2. Changez le Statut de "Actif" a "Desactive"
  3. Cliquez "Enregistrer"
  4. Le client ne pourra plus se connecter — il verra un message "Compte desactive"
Reversible : Pour reactiver, remettez le statut a "Actif" et re-enregistrez.

17. Questions frequentes (FAQ)

Un client a perdu son telephone, que faire ?

Pas de panique ! Les donnees sont sauvegardees dans Firebase (cloud). Le client n'a qu'a ouvrir son lien personnalise sur un nouveau telephone, entrer son code d'acces, et toutes ses donnees se rechargent automatiquement (eleves, paiements, historique complet).

Un client veut utiliser l'app sur 2 telephones ?

C'est possible ! Les donnees se synchronisent automatiquement entre les appareils via Firebase. Un eleve inscrit sur le telephone A apparaitra sur le telephone B des que les deux ont internet.

Comment mettre a jour l'application ?

Il suffit de pousser les modifications sur GitHub. Netlify deploie automatiquement. Grâce au Service Worker en mode "network-first", les clients verront la nouvelle version au prochain chargement avec internet.

L'hebergement est-il vraiment gratuit ?

Oui. Netlify offre un plan gratuit largement suffisant pour cette application (100 Go de bande passante/mois, deploiements illimites). Firebase a aussi un plan gratuit (Spark) qui couvre nos besoins.

Combien de clients peut-on gerer ?

Il n'y a pas de limite technique. Chaque client est un simple fichier JSON de quelques Ko. Vous pouvez gerer des centaines d'ecoles sans probleme.

Est-ce que le code source est securise ?

Le depot GitHub est public (le code est visible), mais les donnees des clients (eleves, paiements) sont stockees dans Firebase Firestore (cloud Google securise) et ne sont jamais sur GitHub. L'acces a Firestore est protege par des regles de securite : seuls les utilisateurs authentifies peuvent lire/ecrire les donnees. Les tokens et mots de passe ne sont pas dans le code source.

Que se passe-t-il si GitHub ou Netlify ferme ?

L'application continuerait a fonctionner hors-ligne sur les appareils ou elle est deja installee. Les donnees restent en securite dans Firebase (independant de GitHub/Netlify). Pour remettre l'app en ligne, il suffirait de re-deployer le code sur un autre hebergeur (Vercel, Cloudflare Pages, etc.).

18. En cas de probleme

ProblemeSolution
Le panneau admin ne se connecte pas Verifiez votre connexion internet. Firebase Auth necessite internet. Verifiez l'email et le mot de passe.
Le deploiement ne fonctionne pas Verifiez que le token GitHub est valide. Regenerez-en un si necessaire (voir section 6).
Le lien du client affiche le mauvais logo Videz le cache du navigateur et rechargez. Verifiez que le fichier JSON du client sur GitHub a le bon logo.

L'impression Bluetooth ne fonctionne pas Verifiez que RawBT est installe, que l'imprimante est appairee en Bluetooth, et que RawBT est configure en mode "ESC/POS".
